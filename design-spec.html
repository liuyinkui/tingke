① 品牌 Logo 方案

方向 A — 纯文字极简

只用文字，不加图标。适合启动屏、App 标题栏、分享卡片。

┌──────────────────────────────────────────────┐ │ │ │ 听　　刻 │ │ TingKe │ │ │ │ 把声音刻进脑子里 │ │ │ └──────────────────────────────────────────────┘
层级内容字体粗细字号字距
L1"听刻" 中文PingFang SCBold (700)28px4px（疏排，呼吸感）
L2"TingKe" 英文SF ProMedium (500)11px3px
L3标语PingFang SCRegular (400)10px1px

设计意图："刻"字本身有雕刻感，疏排字距模拟"刻字"的工整间距。不加修饰，干净自信。

方向 B — 图文结合（推荐）

圆形印章 + 声波图案 + "听刻" 文字并排。App 图标（iOS/Android）、登录页、所有品牌触点使用。

┌──────────────────────────────────────────────┐ │ │ │ ┌──────────┐ │ │ │ ／＼／＼ │ 听刻 │ │ │ ＼／＼／ │ TingKe │ │ │ ● 刻印 │ │ │ └──────────┘ │ │ ^ 印章图标 │ │ │ └──────────────────────────────────────────────┘ 印章图标描述： ┌─────────────────────────────────────┐ │ 外圈圆环 — 深蓝 #1E3A5F │ │ 内部声波 — 五根竖线，中间最高 │ │ 中间圆点 — 暖橙 #FF6B35（刻痕感） │ │ 整体概念：圆形印章 + 声波 = "刻" │ └─────────────────────────────────────┘

推荐理由：印章是"刻"最直接的视觉隐喻，声波代表"听"。两者合在一起就是"听刻"——不需要文字解释，看一眼就懂。

方向 C — 极简符号（小尺寸专用）

纯符号，无文字。用于 App 图标角标、通知、favicon、头像等极小尺寸场景。

Token色值色样PRD 定的我的意见

交互逻辑（王工重点关注）

跟所有其他 App 不一样的地方：
交互逻辑为什么这样设计
逐句听写 音频按句分割 → 用户写完当前句 → 自动播下一句
每句完成后立即显示对/错结果		 即时反馈让用户有游戏化的掌控感，而不是"写完一整段等判卷"的考试感
留空可跳过 听不出的词直接留空（点击空格按钮），继续下一句 不卡住用户的节奏——"不会就是你的学习价值"
点击"再听一遍" 播放当前句原文，不跳到句首 用户只是想再听一次确认，不是要重新开始
提交后不可修改 用户提交当前句后，结果固定，进入下一句 防止纠结，保持节奏
全句听写完成后 显示本段完整结果 → 点击"进入跟读" → Step 3 听写的终点 = 跟读的起点

视觉反馈规则

✅ 正确 (绿色) · 词框背景: #E8F8F5 · 文字颜色: #00B894 · 状态: 用户写对了，不显示原词（因为就是正确的） · 动效: 一个轻微的"闪光"（300ms opacity 闪一下） ❌ 错误 (红色) · 词框背景: #FFF5F5 · 文字颜色: #FF6B6B · 用户写的错误文字加删除线 · 下方显示正确答案（红色加粗） · 动效: 轻微抖动（shake animation, 200ms） · 这个词自动加入"盲区词汇" ⚪ 留空 (灰色) · 词框背景: #F5F7FA · 文字颜色: #B2BEC3 · 显示 "___" 表示用户没听出来 · 下方显示正确答案（灰色） · 这个词自动加入"盲区词汇"

听写输入控件说明

每个"词槽"是一个独立输入框： ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐ │Today │ │ ___ │ │going │ │ ___ │ └──────┘ └──────┘ └──────┘ └──────┘ ✓自动 ⚪留空 ✓自动 ⚪留空 行为： · 用户输入一个词 → 按空格或 Tap Tab → 跳到下一个词槽 · 词槽宽度根据预期词长：最短 40px，最长自适应（但不超过120px） · 正在输入的词槽有底部边框 #1E3A5F 2px 高亮 · 空词槽底部边框 #DFE6E9 1.5px · 用户点击一个空词槽 → 自动播放当前句（辅助听写）
⚠️ 特别注意：
  • 听写时绝对不显示原文——人类会忍不住看答案，训练就废了。
  • 听写完成前，底部"上句结果"区域只显示上一句的结果，不显示所有句的结果（防止用户看前面的答案推导后面的内容）。
  • 用户中途退出 → 弹窗确认"退出后当前进度将丢失"，并提供"保存草稿"选项（仅保存已写对的词）。

原声/我的对比

两个按钮组成 Segmented Control： ┌─────────┬──────────────────┐ │ 🔊 原声 │ 🗣️ 我的录音 │ └─────────┴──────────────────┘ 选中态：背景 #1E3A5F · 白色文字 未选态：透明背景 · 灰色文字 #636E72 点击切换时： · 如果"我的录音"有多次 → 默认播放最新一次 · 用户也可以通过"我的录音"内的下拉选择历史录音 · 播放过程中切换 → 当前播放停止，切换目标开始播放 波形同步高亮： · 播放到哪，原文文字高亮到哪（类似卡拉OK歌词效果） · 对比时特别有用：一眼看出"原声在这里停顿了，但我没有"
⚠️ 给王工的提醒：
  • 录音时长上限 10 秒——太长的录音会增加服务端延迟，用户等不了
  • 如果用户一句话超过 10 秒 → 自动截断，按已有部分评分
  • 评分延迟控制在 2 秒以内，否则用户会反复按录音按钮
  • 录音前检查麦克风权限，第一次请求时说明"需要麦克风权限来评估您的发音"

  • 无底部 Tab Bar： 首页、素材库、个人中心之间用导航 push/pop，不 tab 切换

  • 字体使用系统默认： 不引入第三方字体，PingFang SC / SF Pro 就是最佳选择
