# 听刻 · 产品需求文档

> 版本：v2.1
> 日期：2026-06-03
> 品牌：听刻 — "把声音刻进脑子里"
> 方法：精听法（精听→听写→跟读）
> 目标用户：备考大学生 + 职场英语提升人群

---

## 1. 产品定位

**一句话：** 一款**基于精听方法论**的英语学习App，让大学生和职场人每天用15分钟，通过精听→听写→跟读的闭环真正提升英语听力。

**核心差异化（跟所有人不一样的地方）：**

```
其他App：给你内容，你自己想办法用
    ↓
我们：给你方法+内容+流程，你打开就练

不是"又一个听力资源库"
而是"每天有教练带着你练精听"
```

**品牌定位词：** 方法型 · 极简 · 严肃有效

**品牌名：听刻**
- 谐音"听课"，但"刻"是雕刻、刻入
- "把声音刻进脑子里" —— 跟精听法的理念完全一致
- 不依赖任何名人，完全独立

**内容营销关键词矩阵：**
```
SEO/搜索时用：                       品牌文案中用：
├── 尚雯婕英语学习方法                  ├── 精听一句胜过泛听十篇
├── 精听训练App                       ├── 把声音刻进脑子里
├── 英语听写App                        ├── 每天15分钟，听懂世界
├── 四六级听力训练                     └── 你的专属精听教练
├── 英语跟读评分
└── 影子跟读法

品牌名不依赖尚雯婕，但搜索流量靠她带。
就算她那边出问题，品牌名是"听刻"，换个方法名一样做。
```

## 2. 核心用户

### 为什么尚雯婕方法天然适合大学生/职场人

```
这个方法需要：
    ✅ 自律 — 成年人比孩子自律
    ✅ 重复 — 成人理解重复的价值
    ✅ 目的性 — 考试/面试/工作驱动力强
    ✅ 碎片时间 — 15分钟就能练一段

这个方法不适合：
    ❌ 儿童 — 坐不住，觉得枯燥
    ❌ 零基础 — 完全听不懂的人无法精听
    ❌ 随便学学 — 方法需要commitment
```

## 3. 竞品分析

### 3.1 有精听功能的直接竞品

| App | 精听 | 听写 | 跟读 | AI评分 | 有方法 | 市场表现 |
|-----|------|------|------|--------|--------|---------|
| **每日英语听力** | ✅ | ✅ | ✅ | ❌ | ❌ | 3000+资源，功能堆砌，用户迷茫 |
| **可可英语** | ✅ 四步法 | ✅ | ✅ | ❌ | ⚠️ 有 | UI老旧，方法不突出，用户认知低 |
| **缤纷英语** | ✅ | ✅ | ✅ | ❌ | ❌ | 几年不更新，濒死产品 |
| **天天听英语** | ✅ | ✅ | ✅ | ✅ AI | ❌ | 新App，偏美剧娱乐 |
| **咕噜口语** | ✅ 三步法 | ❌ | ✅ | ✅ | ⚠️ 有 | 偏口语，不是精听 |
| **Parroto（海外）** | ✅ | ✅ | ✅ | ✅ | ✅ 听写→跟读 | **最接近我们**，但面向全球不做中文市场 |
| **一心复读机** | ✅ 播放工具 | ❌ | ❌ | ❌ | ❌ | 纯工具，没有内容和方法 |

### 3.2 间接竞品（用户重叠但方向不同）

| App | 定位 | 跟我们不冲突的原因 |
|-----|------|------------------|
| **百词斩** | 背单词 | 跟精听是两个场景，用户可同时用 |
| **流利说** | AI口语对话 | 偏输出，我们是输入打底 |
| **多邻国** | 游戏化入门 | 太浅，认真学英语的人觉得不够 |
| **HelloTalk** | 真人语伴 | 社交属性，不是练听力 |
| **扇贝** | 阅读+单词 | 偏阅读，不同赛道 |
| **星火英语** | 真题题库 | 刷题用，不是练能力 |

### 3.3 核心发现：市场有四个空白

| # | 空白 | 说明 |
|---|------|------|
| 1 | **没有方法型App** | 所有App都是工具/资源型，没人告诉用户"每天该做什么" |
| 2 | **精听体验都很糙** | 有听写功能的App，交互都停留在十年前（文本框+提交按钮） |
| 3 | **没有"尚雯婕"概念的产品** | 小红书/B站搜这个关键词量很大，但没有App接住流量 |
| 4 | **大而全 vs 小而美** | 竞品都在堆功能、堆内容，没人做减法 |

### 3.4 我们的机会

```
用户想要什么 → 一个简单到不用想"今天练什么"的App
    ↓
现有App给什么 → 3000+资源、10种模式、20个设置项
    ↓
我们给什么 → 每天1段素材、3步走完、15分钟
    ↓
差距 = 机会
```

## 4. 核心功能（MVP）

### 4.1 每日学习流（唯一主路径）

这是App的**全部核心**。用户打开App只做一件事：

```
每天打开App
    │
    ├── Step 1: 盲听  🎧
    │    └── 听一遍今天的素材，不看文字，试试听懂多少
    │        内置复读控制：单句循环、AB段循环、0.5x~1.5x变速
    │
    ├── Step 2: 听写  ✍️
    │    └── 边听边打字，写不出的留空
    │        系统自动比对原文，标记：
    │            ✅ 正确 = 绿色
    │            ❌ 错误 = 红色（显示正确答案）
    │            ⚪ 留空 = 灰色（没听出来）
    │
    ├── Step 3: 跟读  🗣️
    │    └── 录音 → AI评分 → 标注发音问题
    │        原声对比播放 → 听到自己差在哪
    │
    └── 完成 ✅
         学习报告：
         · 今日辨音准确率
         · 新掌握的词汇
         · 需要复习的盲区
         · 连续打卡 streak
```

**关键原则：没有"选择"。** 每天打开就是在唯一路径上。练完收工。

### 4.2 素材库（浏览+选择）

但用户偶尔也想自己挑想学的：

```
素材库
├── 推荐给您的（基于水平和历史表现）
├── 按难度
│   ├── L1 VOA慢速
│   ├── L2 VOA常速 / CET-4
│   ├── L3 CET-6 / 考研
│   ├── L4 TED / 雅思
│   └── L5 经济学人 / 深度访谈
├── 按场景
│   ├── 四六级冲刺
│   ├── 考研英语
│   ├── 职场英语
│   └── 日常口语
└── 按来源
    ├── VOA
    ├── CET真题
    └── 更多（后续扩充）
```

### 4.3 个人中心

```
个人中心
├── 学习统计
│   ├── 累计学习天数
│   ├── 累计听写正确率曲线
│   ├── 词汇量估算
│   └── 本周学习时长
├── 我的单词本
│   ├── 听写中写错的词（自动收录）
│   ├── 收藏的单词
│   └── 按日期查看盲区
├── 设置
│   ├── 每日目标（15/30/45分钟）
│   ├── 通知提醒时间
│   ├── 当前水平（L1~L5）
│   └── 英音/美音偏好
└── 关于/分享
```

### 4.4 声音键（可选参与）

用户在App首页附近的某个入口可以选一种自己喜欢的"学习陪伴声音"：
- 白噪音（雨声、海浪、篝火…类似潮汐）
- 专注计时器（番茄钟风格，听写时倒计时走完提醒休息）
- 完成后的简短鼓励音效

可能不是MVP的必要功能，但在精听的高强度环境下，一个有陪伴感的听觉氛围能改善枯燥感。

### 4.5 第一期不做清单

```
❌ 付费系统（先全免费，跑通再定价）

❌ 老师端/班级功能
❌ 直播/视频课
❌ 家长端
```

|--------|------|------|------|------|

### 内容规格

```
每段音频长度：15秒 ~ 2分钟
    → 太短不够练，太长会让初学者崩溃
    → 严格控制在15秒内（L1）到2分钟（L4）

逐句时间轴：精确到每一句的开始和结束时间
    → 这是听写功能的基础
    → 我可以用脚本自动生成

每篇配套：
    ├── 原文文稿
    ├── 翻译（后续版本加）
    ├── 重点词汇列表
    └── 难度标签
```

## 7. 产品设计 & 用户体验

### 7.1 设计哲学：三句话

| # | 原则 | 解释 | 对应设计决策 |
|---|------|------|------------|
| 1 | **减法优先** | 每个功能存在的前提是：没有它用户会不会死？不会就去掉 | 首页只有一个按钮"开始今日练习" |
| 2 | **零思考** | 用户打开App不需要想"点什么"，路径只有一条 | 没有菜单、没有tab栏、没有选择恐惧 |
| 3 | **进度感** | 用户永远知道自己在哪、完成了多少、离结束还有多久 | 进度条、完成度、倒计时 |

### 7.3 关键交互细节

**音频播放器（核心组件）：**
```
不是系统播放器套个壳

必须有的：
    ├── 波形图可视化（看到声音的起伏）
    ├── 点击波形任意位置跳转
    ├── 单句循环（自动停在句首）
    ├── AB段循环（选中一段反复听）
    ├── 0.5x / 0.75x / 1x / 1.25x / 1.5x 变速
    └── 当前进度 + 总时长

绝对不要：
    ❌ 复杂的设置菜单
    ❌ EQ均衡器
    ❌ 歌单式播放列表
```

**听写交互（跟其他App不一样的地方）：**
```
其他App的听写：
    文本框 → 打一整段 → 提交 → 系统判对错
    → 体验像考试，枯燥

我们的听写：
    逐句听写 → 每句独立 → 每句即时反馈
    → 输完一句立刻知道对错 → 错的不纠结，下一句继续
    → 体验像玩游戏过关

重点：
    不要求"全部写对才能继续"
    写错没关系 → 标记盲区 → 进入跟读阶段
    盲区就是你的学习价值
```

**跟读录音：**
```
类似微信语音消息的交互
    → 长按录音 → 松手上传 → 秒出评分
    → 不要求用户"点开始 → 点停止 → 点提交"
    → 2秒完成录音
```

**完成页（心理设计）：**
```
这是最重要的页面
    → 用户在这里决定明天还来不来

设计原则：
    · 强烈的完成感（🎉 动效）
    · 看到进步（比昨天高5%）
    · 看到连续性（🔥 streak）
    · 低门槛离开（没有"分享到朋友圈"的强制）
```

### 7.4 视觉风格

```
调性：专业 · 安静 · 不花哨

配色：
    主色：深蓝 #1E3A5F（沉稳、专业）
    强调：暖橙 #FF6B35（进度、打卡）
    背景：浅灰 #F5F7FA（干净、不刺眼）
    文字：深灰 #2D3436（可读性）
    正确：绿色 #00B894
    错误：红色 #FF6B6B

字体：
    中文：系统默认（San Francisco / PingFang）
    英文：系统默认（SF Pro / Roboto）
    → 不引入第三方字体，加载快、原生感强

动效：
    克制使用
    ✅ 完成页的庆祝动效
    ✅ 打卡/streak更新
    ❌ 页面切换动画
    ❌ 卡片飞入飞出
    ❌ 粒子/烟花效果

整体感觉：
    像"潮汐"（冥想App）的安静感
    配"多邻国"的进度感
    去掉"多邻国"的幼稚感
    去掉"潮汐"的过于极简
```

### 7.5 为什么这样设计

| 设计选择 | 对应什么用户心理 |
|---------|----------------|
| 首页只有一个按钮 | "别让我想，告诉我该做什么" |
| 学习流分3步，永远显示进度 | "我知道还要多久，我不焦虑" |
| 听写逐句即判 | "即时反馈让我有掌控感" |
| 完成页展示进步曲线 | "我花钱不是为了功能，是为了结果" |
| 不强制分享 | "尊重我，我才会推荐你" |
| 没有tab栏 | "一个App不需要有10个入口" |

### 7.6 需要避免的设计陷阱

```
❌ 首页放"推荐素材列表"
    → 用户打开会想"今天听哪个？" → 犹豫 → 关掉App

❌ 听写时显示原文
    → 人类会忍不住看答案 → 失去训练意义

❌ 太多设置选项
    → 一个英语App不需要"通知声音选择"和"主题色切换"

❌ 完成页弹出"购买VIP"
    → MVP阶段别恶心用户

❌ 跟微博/抖音一样的feed流
    → 这是工具不是媒体
```

### 单用户账本

**免费用户：**
```
收入： ¥0
成本： ¥100/月服务器 ÷ 总注册用户
     = ¥100 ÷ 150（第3个月估算）≈ ¥0.67/用户/月
毛利： -¥0.67/月
```

**付费用户（按月订阅¥29）：**
```
月收入： ¥29
苹果扣25%： ¥29 × 0.75 = ¥21.75
平均付费时长： 5个月（月流失20% → 平均存活5个月）
单用户LTV： ¥21.75 × 5 = ¥108.75
一个付费用户赚回来的钱，可以覆盖162个免费用户的服务器成本。
```

## 13. 2周留存钩子

精听法到第14天会撞上一堵墙：

```
第1天：新鲜感
第3天：觉得有点用
第7天：开始烦躁（同一段听好几遍）
第14天：⚠️ 想放弃了

用户心里在想：
    "我都练了14天了，听力真的进步了吗？"
    "好像没什么感觉…"
    "明天不来了也没关系吧"
```

### 钩子1：第14天前后对比（核心）

用户练了14天后，打开App弹出一张卡片：

```
┌─────────────────────────┐
│                         │
│   🎉 你已练习 14 天！     │
│                         │
│   第一天听写正确率：32%    │
│   最近一天正确率：71%      │
│                         │
│   ▓▓▓▓▓▓▓▓▓░ +39%       │
│                         │
│   "你看，真的有进步"      │
│                         │
└─────────────────────────┘
```

**技术实现：** 第一天就存好用户的首次听写成绩，第14天自动展示对比，不需要任何额外用户操作。

### 钩子2：Streak的"舍不得"效应

人投入越多越难放弃。14天是一个临界点：

```
7天内断了   → "算了重新来"
14天以上断了 → "好可惜！"
```

第14天设计一个"Streak守护"提示：

```
┌─────────────────────┐
│  🔥 14天连胜         │
│                     │
│  你今天还没练        │
│  再练一天就满15天 →  │
│  额外奖励：解锁新素材 │
└─────────────────────┘
```

### 钩子3：第15天解锁"你的盲区专练"

```
系统基于前14天的听写错误数据
自动生成一段"你最容易听错的词汇"专项音频
→ 用户觉得"这个App真的懂我"
→ 个性化本身就是留存力
```

### 三个钩子的节奏

```
第1-7天：streak 硬撑（习惯养成期）
第14天：前后对比（看见进步，情感冲击）
第15天：个性化盲区素材（App懂我，继续依赖）
```
